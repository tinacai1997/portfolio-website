Creating the initial structure for my portfolio website.

I began my website by writing HTML documentation using the IDE, Atom. To ensure accessibility, I used a mobile-first approach. To focus on content over style, I referenced sample wireframes.

CSS

Designing wireframes to guide my website design

As I was writing CSS to emulate the sample wireframes, I noticed I was constantly tweaking the style. So, I decided to create wireframes to guide my design.

To ensure accessibility across devices, I followed a mobile-first approach. Seeing my design from the development perspective revealed the importance of creating clear and consistent assets as a designer.

JavaScript

Adding logic and personality to my website for increased interactivity

I created a JavaScript file and added jQuery to my HTML documents to add advanced interactivity to my site, like a menu button.

Iteration

Most test participants thought my site design was too plain.

I conducted five usability tests to assess my site's usability. Participants had no issue with the test scenarios, but all commented on my site being too plain.

To address this, I added dynamic page colors and parallax images to add dynamics to my portfolio. These additions were well received when critiquing my site with other designers.

Lessons

  1. I learned that efficient development starts with a solid design, even if an assignment doesn't ask you for one.
  2. Since this project was my first attempt at developing a website, I hand coded most of it for practice. I'd likely use frameworks in the future to say the least.
  3. I initially used Atom per assignment instructions but eventually switched to VS Code as I was more familiar with it. This helped me focus on learning one thing at a time.
