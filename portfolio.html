Design Portfolio

Creating a home for my project work

Overview

How can I share my UX work, skills, and experiences with others?

My design portfolio is an online platform that displays my work, design skills,and personality to other designers, potential recruiters, and hiring managers.
This project also serves as a medium for me to practice and demonstrate my understanding of front-end development and web design.
Role: Front-End Dev, Web/UX/UI Designer (solo)
Tools: Adobe XD, VS Code, Atom, GitHub, Google Sheets
Timeframe: Nov 2021 - Present

HTML

Creating the initial structure for my portfolio website.

I began creating my website by writing its HTML documentation using the IDE, Atom. To ensure accessibility, I coded my portfolio using a mobile-first approach. In order to focus on content over style during this stage, I based my web structure on sample wireframes.

Sample Wireframes Structure

GITHUB

Using a repository to track my work.

I used GitHub Desktop to create a repository for my website. I anticipate when I work on team projects, GitHub will be even more useful for reviewing and working on shared files, so learning to use it during this project felt like good practice.
Visit Project GitHub

CSS

Taking a new style direction.

With the structure finished, I began writing the CSS to emulate the sample wireframes. However, I noticed I was constantly tweaking the CSS as I was writing. Therefore, I decided I needed something to guide my design the way I wanted it.

Sample Wireframes Style

DESIGN

Designing wireframes to guide my responsive web design.

To create a guideline for my web design, I sketched and annotated lo-fi wireframes ideas for my portfolio. Then, I translated my sketches to hi-fi wireframes. It was important to keep my site usable for a variety of devices, so I created both mobile and x-large wireframes then adjusted the CSS accordingly. Seeing my design from the development perspective revealed the importance of creating clear and consistent assets as a designer.

MICRO INTERACTIONS + ANIMATION

Adding movement and personality to my website.

I explored CSS interactions and animations to add more visual flair to my site. It was important that these iteractionis conveyed meaning as well. To avoid distracting movement, I animated some elements on hover or click (try to find the easter egg animations hidden throughout my site! ðŸ•µ)

JAVASCRIPT

Developing the website for increased interactivity.

Micro-interactions are fun and useful, but don't provide logic-based functionality. I created a JavaScript file and added jQuery to my HTML documents to add advanced interactivity to my site. I wrote a script for a menu to help users traverse my portfolio. Press the menu icon to check it out! ðŸ‘‰
I also used a plugin called "Slick" to create slideshows for displaying my project content.

TESTING + ITERATION

Most test participants thought my site was too plain.

I conducted five usability tests to observe whether participants could learn about my projects, learn about me, find my resume, and contact me using my site. Test participants had no issue completing the test scenarios, but all commented about the site being too plain.
To address this, I added dynamic page colors and parallax images to add style and movement to my portfolio. These additions were well received when critiquing my site with other designers!

CONCLUSION

What did I learn from my first hand-coded product.

This online portfolio is the first product I've fully developed myself and is a constant work in progress! This project was a great way for me to practice developing and UX projects storytelling ðŸ˜„.
  1. Creating wireframes for design work. Working with sample wireframes would have gotten my portfolio ready earlier, but I had prior-experience with development and didn't want to create a site that I'd eventually change entirely. Next time I work on a website, I know to work on wireframes ahead of time.
  2. Hand coding vs. frameworks. Since this is my first time working on an official website, I hand coded most of it for practice. I know this isn't practical most of the time - I'd likely use frameworks in the future.
  3. Work with what you're used to (unless you can't). I initially used Atom to follow the sample wireframe instructions. I eventually switched to VS Code since I had used it before and was more familiar with it. This allowed me to focus on learning one thing at a time. (VS Code also has the Live Server extension, which is truly a life saver).
